# 战略性压缩技能

建议在工作流程中的关键点手动执行 `/compact`，而不是依赖任意的自动压缩。

## 何时启用

- 运行接近上下文限制（200K 个标记）的长会话时
- 处理多阶段任务（研究 → 计划 → 实施 → 测试）
- 在同一会话中切换不相关任务时
- 完成重要里程碑后开始新工作时
- 当响应变慢或连贯性下降（上下文压力大）时

## 为什么采用战略性压缩？

自动压缩会在任意点触发：
- 常在任务中途，导致重要上下文丢失
- 无法识别逻辑任务边界
- 可能打断复杂的多步骤操作

在逻辑边界处进行战略性压缩：
- **探索后，执行前** — 压缩研究上下文，保留实施计划
- **完成里程碑后** — 为下一阶段工作提供全新起点
- **重大上下文切换前** — 清理探索上下文以便处理不同任务

## 工作原理

`suggest-compact.js` 脚本在 PreToolUse（编辑/写作）时运行，并：

1. **跟踪工具调用** — 统计会话中工具调用次数
2. **阈值检测** — 在可配置阈值触发建议（默认：50 次调用）
3. **定期提醒** — 阈值后每 25 次调用提醒一次

## 钩子设置

添加到你的 `~/.claude/settings.json`：
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}
```
## 配置

环境变量：
- `COMPACT_THRESHOLD` — 在第一次建议之前的工具调用次数（默认值：50）

## 压缩决策指南

使用此表决定何时进行压缩：

| 阶段转换 | 是否压缩？ | 原因 |
|-----------------|----------|-----|
| 研究 → 计划 | 是 | 研究内容庞大；计划是精炼后的输出 |
| 计划 → 实施 | 是 | 计划在 TodoWrite 或文件中；释放上下文以进行编码 |
| 实施 → 测试 | 可能 | 如果测试引用最近的代码则保留；如果转换关注点则压缩 |
| 调试 → 下一个功能 | 是 | 调试跟踪会污染与当前工作无关的上下文 |
| 实施中 | 否 | 丢失变量名、文件路径和部分状态代价高 |
| 失败的方法之后 | 是 | 在尝试新方法前清除死胡同的推理 |

## 压缩后仍保留的信息

了解哪些信息会保留有助于你自信地进行压缩：

| 保留 | 丢失 |
|----------|------|
| CLAUDE.md 指令 | 中间推理和分析 |
| TodoWrite 任务列表 | 之前读取的文件内容 |
| 内存文件 (`~/.claude/memory/`) | 多步对话上下文 |
| Git 状态（提交、分支） | 工具调用历史和次数 |
| 磁盘上的文件 | 用户口头表达的细微偏好 |

## 最佳实践

1. **计划后压缩** — 一旦计划在 TodoWrite 中确定，就进行压缩以重新开始
2. **调试后压缩** — 在继续之前清除错误解决的上下文
3. **实施中不要压缩** — 保留相关更改的上下文
4. **阅读建议** — hook 告诉你*何时*，你决定*是否*
5. **压缩前写入** — 在压缩前将重要上下文保存到文件或内存中
6. **使用 `/compact` 并附带摘要** — 添加自定义消息：`/compact Focus on implementing auth middleware next`

## 令牌优化模式

### 触发表懒加载
在会话开始时不要加载完整技能内容，而是使用将关键字映射到技能路径的触发表。只有在触发时技能才会加载，从而将基线上下文减少 50% ：

| 触发 | 技能 | 加载时 |
|------|------|--------|
| "test", "tdd", "coverage" | tdd-workflow | 用户提到测试时 |
| "security", "auth", "xss" | security-review | 与安全相关的工作 |
| "deploy", "ci/cd" | deployment-patterns | 部署相关上下文 |

### 上下文组合意识
监控什么在消耗你的上下文窗口：
- **CLAUDE.md 文件** — 始终加载，保持精简
- **已加载技能** — 每个技能增加 1-5K 令牌
- **对话历史** — 随每次交流增长
- **工具结果** — 文件读取、搜索结果增加负载

### 重复指令检测
重复上下文的常见来源：
- `~/.claude/rules/` 和项目 `.claude/rules/` 中相同的规则
- 重复 CLAUDE.md 指令的技能
- 涉及重叠领域的多个技能

### 上下文优化工具
- `token-optimizer` MCP — 通过内容去重实现自动 95% 令牌减少
- `context-mode` — 上下文虚拟化（示例从 315KB 降到 5.4KB）
